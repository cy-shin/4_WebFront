익명 함수(이름이 없는 함수)

        
            - 함수 구문
            function([매개변수]){ // 1.함수 선언(함수명 없음)
                // 2. 함수 정의
            }
        
        
        [익명함수]
        - 이름이 없는 함수
        - 이름이 없어서, 필요 시 마음대로 호출하는 것이 불가능함
        - 이벤트 핸들러처럼 선언된 함수가 여러 곳에서 재사용되는 것이 아니라...
          '특정 동작이나 상황에서 수행되는 함수'가 필요할 경우에 사용!

          + 매개변수로 전달되는 함수의 형태가 일정하지 않을 때에도 활용!

즉시 실행 함수

        
            1. ()() // 소괄호 2개
            2. (function([매개변수]){ // 왼쪽 괄호에 익명함수를 작성
                // 함수 정의
            })(); 
        

        - 익명 함수의 한 종류로써, 
        정의 되자마자 바로 실행되는 함수

        - Q. 왜 함수 호출 구문을 정적 요소로 쓰는 방법 대신 즉시 실행 함수를 쓸까?
        - A. 함수 선언 및 호출을 위한 탐색 과정이 생략되어 속도 측면에서 우위를 가짐
        - A. 또한, 변수명 중복을 방지할 수 있다
             (즉시 실행 함수 내에 변수가 작성되면 지역변수로 분류되므로, 
              다른 지역의 함수와 중복될 일 없음)

화살표 함수

        익명 함수의 표현식을 간단히 표현한 표기법. (es6)

        작성법 1. 기본 형태   ([매개변수]) => { 함수 정의 }

            익명 함수   : function(){ }
            화살표 함수 : () => { }

            익명 함수   : function(num){   return num * 2;   }
            화살표 함수 : (num) => {  return num * 2;  }

            ** function을 없애고, 
               소괄호(매개변수 자리)와 중괄호(함수 정의문) 사이에 화살표 넣기

        
        작성법 2. 매개변수가 "하나"인 경우 () 생략 가능

            익명 함수   : function(e){  e.target.style.backgroundColor = "yellow"; }
            화살표 함수 : e => { e.target.style.backgroundColor = "yellow"; }


        작성법 3. 매개변수가 없을 경우 "무조건" () 작성

            익명 함수 : function(){...}
            화살표 함수 : () => {...}

        작성법 4. 함수 정의 부분이 return [식 또는 값] 으로만 작성되어 있는 경우
                {}, return 생략 가능

                익명 함수      : function(num){ return num * 2; }
                화살표 함수    : (num) => num * 2
                (작성법2 추가) : num  => num * 2


        작성법 5. 함수 정의 부분이 return 구문만 있으나
                return 되는 값이 객체(object)인 경우 {}, return 생략 불가능

                익명 함수   : function(){ return {"name":"홍길동", "age":20} }
                화살표 함수 : () => { return {"name":"홍길동", "age":20} }
                                    // 객체 반환 시 return 생략 불가능
